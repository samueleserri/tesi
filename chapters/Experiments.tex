\chapter{Numerical Experiments} \label{Experiments chapter}

In this chapter, we run numerical simulations to compare the performance of the different algorithms described in chapter \ref{ALGO CHAPTER}, in terms of convergence speed and relative reconstruction error.
As error measure, we use the relative reconstruction error with the Frobenius norm. This is defined for the $t$-th iteration as:
\begin{equation}\label{relative fro error}
    e_t = \frac{||V - W_t H_t||_F^2}{||V||_F^2}
\end{equation}
The simulations are run with Python 3.14 on a Intel Core™ i7-1260P processor. For the documentation and code availability refer to the ``Code Documentation'' chapter in the appendix \ref{code documentation appendix} of this thesis.
\par
Following the literature, specifically the approach of N. Gillis \cite{gillis2020nonnegative}, we distinguish between the dense and the sparse case.
As a benchmark for the dense case we use the CBCL dataset and the Olivetti faces dataset. For the sparse case we use the tdt2 dataset.

In section \ref{init comparison section} the HALS algorithm is used to compare the influence of the initialization strategy on the convergence speed and the final relative reconstruction error, more specifically we fit the same NMF model on the same data changing only the initialization method.

\section{The CBCL dataset}\label{CBCL dataset section}
For this section, we use the CBCL face dataset used in \cite{Lee1999}, see subsection \ref{Example feature extraction NMF} for more details.
We fit two NMF models with reconstruction rank $r = 10,49$ using the MU algorithm with Frobenius norm ($\beta = 2$), the hierarchical alternating least squares (HALS) algorithm, the alternating least squares (ALS) algorithm, and the projected gradient descent (PGD) algorithm.
Recall from section \ref{initialization section} that the initialization of the matrices $W$ and $H$ can have a significant impact on the convergence and performance of the algorithms.
In order to reduce the variance of the results due to random initialization, we run each algorithm 10 times with 10 different random initializations and we average the relative reconstruction errors over those runs.
We let each algorithm run for 30 seconds.
The results are shown in Figure \ref{fig:CBCL dataset}.
\begin{figure}[htbp]
    \centering
    \includegraphics[width=1\textwidth]{Imgs/Chapter 5/CBCL Low rank.png}
    \includegraphics[width=1\textwidth]{Imgs/Chapter 5/CBCL_random_init.png}
    \caption{Average relative reconstruction error for different algorithms averaged over 10 random initializations on the CBCL dataset. Above the model with $r = 10$, below with $r = 49$}
    \label{fig:CBCL dataset}
\end{figure}

As expected the HALS algorithm converges the fastest and reaches the lowest relative reconstruction error within $30$ seconds, while the ALS algorithm performs the worst, this is a known result in the case of dense matrices, see \cite{gillis2020nonnegative} section 8.4.3.
On the other hand, the PGD and MU have similar performances. 
\par
By looking at the average number of iterations and the average time for iteration, shown in Tables \ref{tab:CBCL results 10} and \ref{tab:CBCL results 49}, we can see that the ALS algorithm has a much higher time for iteration compared to the other algorithms.
While the HALS algorithm is the one that achieve the highest number of iterations within the time limit.
\par

The reconstruction rank is set to $r = 30$ producing two matrices $W \in \R^{4096 \times 30}$ and $H \in \R^{30 \times 400}$.

\section{Comparison between random and NNDSVD initializations}\label{init comparison section}
Finally, we wonder whether we can observe a significant difference in the convergence speed and the final relative reconstruction error if we choose the NNDSVD initialization strategy described in Algorithm \ref{NNDSVD} instead of the random initialization.
We decide to run this test with the HALS algorithm.
As data matrix we use the CBCL dataset and we fit a NMF model with reconstruction rank equal to $49$ using the HALS algorithm as solver with two different initialization strategies:
\begin{enumerate}
    \item random initialization: $W \in \R^{m \times r}$ and $H \in \R^{r \times n}$ are initialized from a uniform distribution in $[0,1)$.
    \item NNDSVD initialization: $W \in \R^{m \times r}$ and $H \in \R^{r \times n}$ are initialized using the NNDSVD method.
\end{enumerate}
Both algorithms are run for 10 seconds. The results are shown in Figure \ref{fig:Hals_init_comparison}.

\begin{figure}[ht]
    \centering
    \includegraphics[width=1\textwidth]{Imgs/Chapter 5/HALS_init_comparison.png}
    \caption{Comparison between random initialization and NNDSVD initialization using the HALS algorithm for fitting a dense matrix.}
    \label{fig:Hals_init_comparison}
\end{figure}

We observe that, in this case, the HALS algorithm initialized with NNDSVD converges faster and reaches a lower relative reconstruction error within the time limit compared to the random initialization.

\section*{Summary}

In this chapter we compared the MU algorithm, the HALS algorithm, the ALS algorithm and the PGD algorithm using the  relative reconstruction error (\ref{relative fro error}). 

For sparse matrices, ALS performs more efficiently compared to dense matrices when it converges poorly.
HALS converges the fastest in both cases. 
MU and PGD have similar performance in the dense case, while in the sparse case MU performs better than PGD.
From tables \ref{tab:CBCL results 10}, \ref{tab:CBCL results 49}, \ref{tab:Top30 news results 30}, and \ref{tab:Top30 news results 60} we can see that the time for iteration of ALS is much higher than the other algorithms in the dense case, while in the sparse case it is lower than PGD and higher than HALS and MU.
In the dense case, HALS is the algorithm that achieves the highest number of iterations within the time limit, while in the sparse case it is the MU algorithm that achieves the highest number of iterations within the time limit.
\par
In general there is high sensitivity to the initialization, sometimes using initialization techniques such as NNDSVD can help performance.
In this section we showed that for the HALS algorithm, the NNDSVD initialization strategy leads to a faster convergence and a lower relative reconstruction error within the time limit compared to the random initialization strategy.
For the MU algorithm, the NNDSVD initialization is usually followed by a procedure that replaces the zero entries of $W$ and $H$ with small positive values or with the average of $V$.
